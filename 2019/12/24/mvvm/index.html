单向数据绑定

正常情况下，更新　module　中的属性，UI并不会自动刷新，需要我们手动设定，但是google 的　DataBinding提供了三种自动驱动界面刷新的类： BaseObservable、ObservableField、ObservableCollection
这三个类都实现了　Observable　接口　允许注册一个监听器，当可观察对象的属性更改时就会通知这个监听器，此时就需要用到 OnPropertyChangedCallback

BaseObservable

该类提供了两种刷新方式：
１、notifyChange()
刷新全部属性
２、notifyPropertyChanged()　
只刷新对应的 BR 的 flag该 BR 的生成通过注释 @Bindable 生成，

ObservableField

继承自 BaseObservableField　是google　官方对　BaseObservable　的封装
官方提供了基本数据类型的封装：ObservableBoolean,ObservableByte,ObservableChar,ObservableShort,ObservableInt,ObservableLong,ObservableFloat,ObservableDouble,ObservableParcelable;
ObservableField 提供了一个泛型，可以用来申请其他类型
eg: mvvm_test/src/main/java/com/wc/mvvm_test

ObservableCollection

对于List集合和map集合　dataBinding　提供了　ObservableList　和　ObservableMap　当其中的数据发生改变时，UI界面也会刷新
eg: mvvm_test/src/main/java/com/wc/mvvm_test

双向绑定数据

即不仅仅module中的数据自动刷新视图，同样视图中的数据改变时，也会更新module．
绑定变量的方式比单向绑定多了一个等号：　android:text="@={goods.name}"

事件绑定

事件绑定也是一种变量绑定，只不过设置的变量是回调接口而已
@{()->buttonClick.BindInfo()}这里用到了Lambda表达式，参数可以自己填，
@{buttonClick.BindInfo}这种方法调用参数和返回值必须和原始的回调函数保持一致

类方法的使用

可以定义一个类，里面提供一些静态方法，在布局文件中就引入该类，就可以再对应的地方使用类方法

运算符

DataBinding 支持在布局文件中使用以下运算符、表达式和关键字

* 算术 + - / * %
* 字符串合并 +
* 逻辑 && ||
* 二元 & | ^
* 一元 + - ! ~
* 移位 >> >>> <<
* 比较 == > < >= <=
* Instanceof
* Grouping ()
* character, String, numeric, null
* Cast
* 方法调用
* Field 访问
* Array 访问 []
* 三元 ?:

此外，DataBinding 还支持以下几种形式的调用

* Null Coalescing
    空合并运算符 ?? 会取第一个不为 null 的值作为返回值
* 属性控制
    可以通过变量值来控制 View 的属性
* 避免空指针异常
    DataBinding 也会自动帮助我们避免空指针异常
    例如，如果 "@{userInfo.password}" 中 userInfo 为 null 的话，userInfo.password 会被赋值为默认值 null，而不会抛出空指针异常

include 和 viewStub

include

对于 include 的布局文件，一样是支持通过 dataBinding 来进行数据绑定，此时一样需要在待 include 的布局中依然使用 layout 标签，声明需要使用到的变量

注意：如果在 xml 中没有使用 bind:ＸＸＸ　对 ViewStub 进行数据绑定则可以等到当 ViewStub Inflate 时再绑定变量，此时需要为 ViewStub 设置 setOnInflateListener回调函数，在回调函数中进行数据绑定
